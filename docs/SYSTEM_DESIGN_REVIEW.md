# System Design Review & Missing Parts

GPS 기반 자동 출퇴근 및 위치 공유 시스템의 완성도를 높이기 위한 추가 검토 사항입니다.

## 1. 보안 및 신뢰성 (Security & Reliability)
- [ ] **GPS 위변조 방지**: 가짜 GPS(Mock Location) 앱 감지 및 서버 측 검증 로직.
- [ ] **이동 경로 타당성 검사**: 이전 좌표와 현재 좌표 간의 이동 속도가 물리적으로 가능한지 체크.
- [ ] **수동 조작 지원**: 자동 시스템 오류(GPS 수신 불량 등)를 대비한 관리자 승인 기반 수동 출퇴근 기능.

## 2. 사용자 경험 (UX & Mobile)
- [ ] **푸시 알림 (Push Notification)**: 자동 출/퇴근 성공, 유예 시간 진입, 위치 공유 요청 등에 대한 즉각적인 알림.
- [ ] **배터리 최적화**: OS 레벨의 Geofence API(Android Geofencing, iOS Region Monitoring) 활용 계획.
- [ ] **오프라인 데이터 동기화**: 네트워크 단절 시 위치 정보를 로컬에 저장 후 복구 시 일괄 전송.

## 3. 프라이버시 (Privacy)
- [ ] **공유 시간대 설정**: 업무 시간 외(퇴근 후, 주말) 위치 공유 자동 차단 기능.
- [ ] **마지막 위치 유효기간**: 사용자가 앱을 끄거나 위치 제공을 중단했을 때, 언제까지의 데이터를 '유효한 최신 위치'로 볼 것인지에 대한 정책.
- [ ] **로그 파기 정책**: 위치 로그 데이터의 보관 기간 설정 (예: 3개월 후 자동 삭제).

## 4. 인프라 및 성능 (Performance)
- [ ] **PostGIS 활용**: 반경 계산 시 단순 수식보다는 DB 레벨의 공간 쿼리(Spatial Query)를 통한 성능 최적화.
- [ ] **Redis 캐싱**: `user_locations` 테이블 대신 Redis의 Geospatial 인덱스를 사용하여 실시간 위치 조회 성능 극대화.
- [ ] **스케줄러 부하 분산**: 자동 퇴근 처리를 위한 스케줄러가 수만 명의 사용자를 동시에 처리할 때의 부하 고려.
